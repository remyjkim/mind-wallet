# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security-sensitive problems.

Instead, report them privately to the maintainers with:

- a description of the issue
- affected package or command
- reproduction steps
- impact assessment
- any suggested mitigation

If the issue involves private keys, wallet custody, payment authorization, or protocol replay risk, include that context explicitly.

## Scope

Security-sensitive areas include:

- private-key handling
- OWS wallet integration
- payment credential generation
- protocol challenge parsing
- MCP server usage in agent environments

## Response Expectations

Maintainers should acknowledge reports promptly, investigate severity, and coordinate a fix and disclosure timeline appropriate to the impact.
